# Contributing to CHAMP

Thank you for your interest in contributing to the CHAMP-EU code. This document provides guidelines and best practices for contributing to the project.

## How to Contribute

### Reporting Issues
- Use the GitHub Issues page to report bugs, request features, or ask questions.
- When reporting a bug, include steps to reproduce, expected behavior, actual behavior, and relevant environment details.

### Submitting Pull Requests
1. Fork the repository.
2. Create a new branch with a descriptive name.
3. Make your changes in a clear and focused manner.
4. Ensure that the code compiles and existing tests pass.
5. Add tests for any new functionality when applicable.
6. Submit a pull request (PR) to the `main` branch.
7. Provide a clear explanation of the changes in the PR description.

### Code Style and Standards
- CHAMP is primarily written in Fortran 2003. Keep code consistent with existing formatting.
- Document significant functions, modules, and algorithms.
- Follow naming conventions already used within the project.

### Testing
- All contributions should be validated using the project's GitHub Actions CI workflows.
- If adding new modules or capabilities, include corresponding test cases where possible.

### Documentation
- Update documentation when changes impact user workflows, input formats, or core functionality.
- Doxygen comments should be used for Fortran source files when appropriate.

### Communication
- Use GitHub Discussions or Issues for technical questions.
- Major changes should be proposed and discussed before implementation.

## Contributor Recognition
We welcome contributions from researchers, developers, and users of CHAMP-EU. Accepted contributions will be credited in the project's release notes.
